# 故障排查

## 常见问题

### 1. 未收到ACK

**原因**: CTC未响应

| 错误模式 | 原因 |
|----------|------|
| RSR错误100% | CTC不响应状态报告 |
| SDCI错误67% | CTC处理能力不足 |
| 所有帧错误 | 网络问题 |

**解决**: 检查物理层、网络连接、CTC应用

⚠️ **不要修改500ms超时**

### 2. DC2/DC3频繁重试

**诊断**:
```bash
grep -E "(DC2|DC3)" ZLEvents260201 | \
  awk '/DC2/{dc2++} /DC3/{dc3++} END{print "DC2:", dc2, "DC3:", dc3}'
```

### 3. SDIQ等待时间长

**解释**: SDIQ每24小时发送一次，正常现象

### 4. 设备状态异常

| 状态值 | 原因 |
|--------|------|
| 0x19 | 位置丢失+占用 |
| 0x07 | 位置冲突 |

## 帧类型分布

| 类型 | 占比 | 说明 |
|------|------|------|
| ACK | 40-50% | 心跳 |
| SDCI | 20-30% | 增量 |
| RSR | 10-15% | 状态 |
| SDI | 5-10% | 全量 |

## 故障等级

| 等级 | 条件 |
|------|------|
| 严重 | ACK错误率>20% |
| 警告 | 设备异常>50个 |
| 提醒 | 双机不一致 |

## 排查命令

```bash
# 统计ACK错误
grep "未收到ACK" ZLEvents* | wc -l

# 统计帧类型
grep -oP '\[[A-Z]+\]' ZLEvents* | sort | uniq -c

# 检查设备索引
grep "^,64," lgxtq.zl
```
