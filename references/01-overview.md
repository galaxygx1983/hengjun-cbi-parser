## 系统架构

### CTC-CBI 通信架构

CTC 调度集中系统，配置两台自律机互为主备，自律机上运行驱动程序，与计算机联锁系统的操作表示机通信。

CBI 计算机联锁系统，配置两台操作表示机互为主备，与调度集中系统的自律机通信。

正常状态下，CTC 自律机与 CBI 操作表示机之间维持交叉通信，共 4 路通信，即：CTC 主机 -- CBI 主机；CTC 主机 -- CBI 备机；CTC 备机 -- CBI 主机；CTC 备机 -- CBI 备机。

### 通信实体

| 实体                    | 角色   | 说明                         |
| ----------------------- | ------ | ---------------------------- |
| **CTC自律机主机** | 主动方 | 发起连接请求，控制数据流方向 |
| **CTC自律机备机** | 主动方 | 维持通信，主机故障时接管控制 |
| **联锁主机**      | 被动方 | 响应请求，报告站场状态       |
| **联锁备机**      | 被动方 | 维持通信，主机故障时接管控制 |

### 日志文件类型

| 日志类型           | 文件格式                           | 记录内容                     | 用途                 |
| ------------------ | ---------------------------------- | ---------------------------- | -------------------- |
| **联锁日志** | `ZLEventsMMDDYY`                 | 联锁系统收发的帧数据         | 帧分析、设备状态追踪 |
| **CTC日志**  | `lgxtcidriver_YYYYMMDD_序号.log` | 连接状态、故障检测、主备切换 | 故障诊断、重连分析   |

### 物理连接规范

| 参数               | 规格                           | 说明         |
| ------------------ | ------------------------------ | ------------ |
| **接口标准** | RS422串行接口                  | 标准工业接口 |
| **通信方式** | 异步全双工                     | 双向同时通信 |
| **光电隔离** | 仅CTC设备侧                    | 安全隔离     |
| **电缆类型** | 屏蔽双绞线或光纤               | 抗干扰       |
| **屏蔽接地** | CTC侧接地，CBI侧悬浮           | 避免地环路   |
| **波特率**   | 19.2k / 38.4k bps              | 可配置       |
| **数据格式** | 8N1 (8数据位, 无校验, 1停止位) | 固定         |

## 核心概念

### 帧类型分类

| 类别               | 帧类型          | 分类       | 说明                                                |
| ------------------ | --------------- | ---------- | --------------------------------------------------- |
| **连接控制** | DC2/DC3         | 通信控制帧 | 建立/确认连接                                       |
| **传输确认** | ACK/NACK/VERROR | 通信控制帧 | 应答/否定应答/协议版本错误                          |
| **数据传送** | SDCI/SDI/SDIQ   | 数据传送帧 | 站场数据变化/完整/请求                              |
| **状态报告** | RSR             | 数据传送帧 | 系统工作状态报告（冗余状态、自律控制/非常站控状态） |
| **故障报告** | FIR             | 数据传送帧 | 故障信息报告                                        |
| **控制命令** | BCC             | 数据传送帧 | 按钮控制命令                                        |
| **时间同步** | TSQ/TSD         | 数据传送帧 | 时间同步请求/数据                                   |

### 设备状态编码

所有设备状态采用**位组合编码**，从低位向高位依次表示各种状态分量。

| 设备类型 | 命名规则 | 示例              |
| -------- | -------- | ----------------- |
| 道岔区段 | 纯数字   | "64", "110"       |
| 信号机   | D+数字   | "D114", "D6"      |
| 无岔区段 | 其他格式 | "DK5", "102/104G" |

### 关键时序约束

⚠️ **以下参数协议固定，不可修改**：

| 参数        | 值     | 说明         |
| ----------- | ------ | ------------ |
| ACK超时     | 500ms  | 协议标准     |
| 最大重试    | 3次    | 原始+2次重传 |
| 中断检测    | 1500ms | 3×500ms超时 |
| SDIQ间隔    | 24小时 | 日同步       |
| 心跳间隔    | 500ms  | 空闲期间     |
| CTC故障切换 | 3秒    | 主备切换超时 |
| CBI故障切换 | 5秒    | 主备切换超时 |

### 可靠性机制

| 机制            | 说明                             |
| --------------- | -------------------------------- |
| CRC校验         | 16位CRC帧尾，数据完整性验证      |
| ACK/NACK/VERROR | 正/负/协议版本错误应答，传输确认 |
| 超时重传        | 500ms超时，最多3次重试           |
| 心跳监测        | 空闲期间周期性ACK，链路监控      |

### 故障切换规则

1. **仅主机-主机故障时切换**

   - CTC主机-联锁主机通信中断 > 3秒 → CTC切换
   - CTC主机-联锁主机通信中断 > 5秒 → 联锁切换
2. **其他路径故障不触发切换**

   - CTC主机-联锁备机
   - CTC备机-联锁主机
   - 备机-备机

## 网络问题处理

### ⚠️ 重要原则

**网络问题必须通过物理层改善解决，不能通过调整超时值解决。**

| 正确做法     | 错误做法      |
| ------------ | ------------- |
| 检查电缆连接 | 增加ACK超时值 |
| 验证接地系统 | 修改重试次数  |
| 测试信号质量 | 调整心跳间隔  |
| 更换网络设备 |               |

### 诊断流程

```
物理层检查 → 链路层检查 → 协议层检查 → 应用层检查
   (电缆/接地)    (波特率/校验)    (帧格式/CRC)     (设备状态)
```
