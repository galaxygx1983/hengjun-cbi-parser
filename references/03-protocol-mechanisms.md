# 协议机制

## 核心约束

⚠️ **ACK超时500ms是协议固定值，不可修改**

网络问题必须通过物理层解决，不能调整超时参数。

## 超时参数

| 参数 | 值 | 说明 |
|------|------|------|
| ACK超时 | 500ms | 固定 |
| 最大重试 | 3次 | 原始+2次重传 |
| 中断判定 | 1500ms | 3×500ms |
| 硬件故障重试 | 6秒 | DC2间隔 |
| CTC切换 | 3秒 | 主机-主机故障 |
| 联锁切换 | 5秒 | 主机-主机故障 |

## 序号机制

### 序号变量

| 变量 | 说明 | 初始值 |
|------|------|--------|
| 发送序号 | 数据帧编号 | DC3后=1 |
| 确认序号 | 确认已收帧 | DC3后=0 |

### 序号控制规则

1. **发送帧时**: 发送序号=发送序号变量，确认序号=确认序号变量
2. **收到ACK**: 确认序号=发送序号 → 发送序号变量+1
3. **未确认**: 不发送新帧，超时重传
4. **收到重复帧**: 确认序号=发送序号，不重复处理
5. **序号差≥2**: 判定通信中断

## 主备通信

### 通信路径

| 路径 | 功能 |
|------|------|
| CTC主机 ↔ 联锁主机 | 完整通信 |
| CTC主机 ↔ 联锁备机 | 仅监控 |
| CTC备机 ↔ 联锁主机 | 查询，禁止控制 |
| 备机 ↔ 备机 | 最小通信 |

### 帧类型限制

- **BCC/TSD/ACA**: 仅CTC主机→联锁主机
- **SDI/SDCI**: 仅联锁主机→CTC主机
- **FIR/ACQ**: 仅联锁主机发送
- **TSQ**: 仅联锁主机发送

## 故障切换

### 触发条件
- **仅主机-主机故障时切换**
- CTC: 中断3秒后切换
- 联锁: 中断5秒后切换

### 切换流程
```
CTC主机故障 → 3秒超时 → 检查备机能否通信 → 切换
联锁主机故障 → 5秒超时 → 检查备机能否通信 → 切换
```

## 故障恢复

| 阶段 | 处理 |
|------|------|
| 检测中断 | 连续3次无ACK(1500ms) |
| CTC响应 | 发送DC2最多3次 |
| 判定故障 | 3次DC2无响应 |
| 恢复模式 | 每6秒发送DC2尝试恢复 |
