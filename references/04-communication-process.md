## 通信流程总览

### 连接建立后的初始状态

DC2/DC3握手完成后：

| 系统               | 初始状态       | 说明                         |
| ------------------ | -------------- | ---------------------------- |
| **控制模式** | 中间状态       | 既不是非常站控也不是自律控制 |
| **序号**     | 发送=1, 确认=0 | 准备数据传输                 |

### 完整通信流程

#### 第一阶段：状态交换（双向）

**双方发送RSR帧：**

| 发送方 | 接收方 | 帧         | 目的                       |
| ------ | ------ | ---------- | -------------------------- |
| CTC    | 联锁   | RSR (0xAA) | 报告CTC主备状态            |
| 联锁   | CTC    | RSR (0xAA) | 报告联锁主备状态和控制模式 |

```
CTC                    联锁
  |-------[RSR]------------>|
  |<------[RSR]-------------|
  |
  [状态交换完成]
```

**关键信息：**

- 双方系统的主/备状态
- 联锁当前的控制模式

#### 第二阶段：初始数据同步（联锁 → CTC）

**联锁发送完整站场数据：**

```
CTC                    联锁
  |<------[SDI]-------------|  (完整站场数据)
  |
  |-------[ACK]------------>|  (确认收到)
  |
  [基线建立完成]
```

**详情：**

- SDI包含所有设备状态
- CTC发送ACK(0x06)确认
- 建立变化追踪的基线

#### 第三阶段：增量更新（持续进行）

**联锁发送状态变化：**

```
CTC                    联锁
  |<------[SDCI]------------|  (变化的设备)
  |<------[SDCI]------------|  (更多变化...)
  |
  [增量更新中]
```

**特点：**

- 站场状态变化时发送SDCI
- 小规模变化比SDI更高效
- 模式转换期间持续发送

#### 第四阶段：自律模式请求（联锁 → CTC）

**步骤1：自检**

- 非常站控模式下无按钮操作执行
- 系统状态稳定

**步骤2：按钮锁定**

- 释放"非常站控"按钮后
- 联锁锁定所有人工按钮（除“非常站控”按钮）

**步骤3：发送ACQ**

- 联锁发送ACQ(0x75)
- 启动5秒超时定时器

```
CTC                    联锁
  |<------[ACQ]-------------|  (请求自律控制)
  |                         | [5秒超时启动]
  |
  [CTC评估请求]
```

#### 第五阶段：CTC决策和响应

**CTC自检：**

1. CTC设备运行正常
2. 非常站控模式下无按钮操作

**决策结果：**

| 条件       | 结果   | ACA值 |
| ---------- | ------ | ----- |
| 全部满足   | 同意   | 0x55  |
| 任一不满足 | 不同意 | 0xAA  |

**响应流程（5秒内）：**

```
CTC                    联锁
  |<------[ACQ]-------------|
  |--[自检]--|
  |--[发送ACA]--|
  |------[ACA 0x55]-------->|
  |                         |
  |                         | [进入自律控制]
```

#### 备选：非常站控模式

若操作员按下**"非常站控"按钮**：

- 立即进入非常站控模式
- 取消任何待处理的自律模式请求

### 完整流程图

```
[开始] - 通信已建立（DC2/DC3）
             |
             ▼
     [初始状态：中间模式]
             |
             ▼
     [发送RSR] ←──────→ [发送RSR]
             |
             ▼
     [发送SDI] （联锁 → CTC）
             |
             ▼
     [等待CTC的ACK]
             |
             ▼
     [发送SDCI] （等待期间发送变化）
             |
             ▼
     [发送ACQ] （请求自律控制）
             |
             ▼
     +-------+-------+-------+
     |               |       |
    非常站控        收到       超时

     |           |

```

## 状态转换详解

### 控制模式状态机

| 当前状态           | 触发条件         | 下一状态           | 备注     |
| ------------------ | ---------------- | ------------------ | -------- |
| **中间状态** | 收到ACA 0x55     | **自律控制** | 正常转换 |
| **中间状态** | 按下非常站控按钮 | 非常站控           | 人工干预 |
| **中间状态** | ACA超时（5秒）   | **中间状态** | 请求失败 |
| **中间状态** | ACA 0xAA         | **中间状态** | 请求被拒 |
| **自律控制** | 系统故障         | 非常站控           | 自动回退 |
| 非常站控           | ACQ + ACA 0x55   | **自律控制** | 可切换回 |

### 时序汇总

| 阶段         | 超时时间 | 说明                |
| ------------ | -------- | ------------------- |
| SDI → ACK   | 500ms    | 必须收到确认        |
| ACQ → ACA   | 5秒      | 必须收到同意/不同意 |
| TSQ → TSD   | 500ms    | 时间同步            |
| 非常站控按钮 | 立即     | 无超时              |

## 通信内容详解

### 1. 站场表示信息

**描述：** 从联锁发送到CTC的状态数据，反映设备状态变化。

**包含内容：**

- **信号机状态：** 灯色、灯丝断、进路切换、延时解锁
- **道岔状态：** 位置（定位/反位）、道岔锁、区段锁/占用
- **区段状态：** 区段锁/占用
- **按钮状态：** 按钮按下状态
- **表示器状态：** 各种表示灯
- **告警信息：** 系统告警和警告

**相关帧：**

- SDI (0x85) - 完整站场状态
- SDCI (0x8A) - 仅变化的设备
- FIR (0x65) - 故障信息报告

### 2. 工作状态信息

**描述：** 系统间运行状态交换。

**包含内容：**

- 联锁系统状态：
  - 主/备运行状态
  - 当前控制模式
- CTC系统状态：
  - 主/备运行状态
  - 同意/不同意自律模式转换

**相关帧：**

- RSR (0xAA) - 双向状态报告

**状态码定义：**

| 值   | 主/备状态 | 控制模式（联锁→CTC） | 控制模式（CTC→联锁） |
| ---- | --------- | --------------------- | --------------------- |
| 0x55 | 主机      | 自律                  | 同意自律              |
| 0xAA | 备机      | 非常站控              | 不同意自律            |
| 0xCC | 未知      | 中间状态              | 不确定                |

### 3. 控制命令信息

**描述：** CTC向联锁发送的操作指令，控制站场设备。

**命令类型：**

| 命令       | 编码 | 说明             |
| ---------- | ---- | ---------------- |
| 调车作业   | 12   | 建立调车进路     |
| 列车作业   | 14   | 建立列车进路     |
| 取消进路   | 4    | 取消已建立的进路 |
| 故障解锁   | 6    | 故障后释放区段锁 |
| 人工解锁   | 3    | 人工延时解锁     |
| 重复开放   | 5    | 重新开放信号     |
| 道岔定位   | 7    | 道岔转至定位     |
| 道岔反位   | 8    | 道岔转至反位     |
| 道岔锁闭   | 9    | 锁闭道岔位置     |
| 道岔解锁   | 10   | 解锁道岔         |
| 关闭信号   | 13   | 立即关闭信号     |
| 引导锁闭   | 16   | 引导信号锁闭     |
| 引导总锁   | 18   | 引导总锁闭       |
| 引导总解锁 | 15   | 释放引导总锁     |

**相关帧：**

- BCC (0x95) - 按钮控制命令

### 4. 时钟信息

**描述：** 系统间时钟同步，CTC提供标准时间。

**流程：**

1. 联锁发送TSQ (0x9A) - 时间同步请求
2. 联锁发送TSD (0xA5) - 当前时间数据
3. 联锁更新本地时钟

**TSD时间数据格式（7字节）：**

| 字节 | 字段 | 格式   | 示例          |
| ---- | ---- | ------ | ------------- |
| 0-1  | 年份 | 小端序 | 0x07CD = 1997 |
| 2    | 月份 | 1字节  | 1-12          |
| 3    | 日期 | 1字节  | 1-31          |
| 4    | 小时 | 1字节  | 0-23          |
| 5    | 分钟 | 1字节  | 0-59          |
| 6    | 秒   | 1字节  | 0-59          |

**相关帧：**

- TSQ (0x9A) - 时间同步请求
- TSD (0xA5) - 时间同步数据

### 5. 心跳信息

**描述：** 无数据传输时交换，表明系统正常运行。

**目的：**

- 表明本地系统运行正常
- 检测网络连通性
- 维护通信会话
- 超过1500ms无有效帧时触发

**机制：**

- ACK帧(0x06)作为隐式心跳
- 收到数据帧时，发送ACK作为确认
- 无数据时，定期发送保活信号
- 超时：1500ms内无有效帧则判定通信中断

**相关帧：**

- ACK (0x06) - 确认/心跳
- DC2 (0x12) - 连接请求（恢复）
- DC3 (0x13) - 连接确认（恢复）

### 6. 控制模式转换信息

**描述：** 控制模式间转换时的数据交换。

**模式定义：**

- **非常站控模式：** 本地人工控制
- **自律控制模式：** 自动CTC控制
- **中间状态：** 模式间转换

**模式转换流程：**

1. **联锁请求自律模式：**

   - 发送ACQ (0x75)
   - CTC评估并回复ACA (0x7A)
   - ACA值：0x55（同意），0xAA（不同意）
2. **CTC允许自律模式：**

   - 发送RSR (0xAA)，控制模式=0x55
   - 联锁切换至自律模式
   - 通过RSR报告新模式
3. **非常站控模式激活：**

- 发生紧急情况时
- 联锁切换至非常站控模式
- 通过RSR报告，控制模式=0xAA

**相关帧：**

- ACQ (0x75) - 自律请求（联锁 → CTC）
- ACA (0x7A) - 自律同意（CTC → 联锁）
- RSR (0xAA) - 状态报告（含控制模式）

## 码位表格式

### 文件结构

码位表文件（lgxtq.zl）包含两个部分：

#### 1. [objects]部分

**用途：** SDCI帧解析的设备索引表

**索引范围：** 从0开始

**格式：** `#,<设备名>,<对象索引>`

**示例：**

```
[objects]
#,D114,63     # D114信号机，对象索引63
#,D116,64     # D116信号机，对象索引64
#,10,100      # 10号道岔区段，对象索引100
```

#### 2. [zlobjects]部分

**用途：** SDI帧解析的字节位置表

**索引范围：** 从1开始

**格式：** `#,<设备名>,<字节索引>,<位偏移>`

**示例：**

```
[zlobjects]
#,10,1,0          # 10号道岔，字节索引1，位偏移0
#,D6,280,0        # D6信号机，字节索引280，位偏移0
#,DK9F,322,0      # DK9F区段，字节索引322，位偏移0（低四位）
#,104/110G,324,4   # 104/110G区段，字节索引324，位偏移4（高四位）
```

### 使用说明

- **SDCI帧：** 使用[objects]表按对象索引查找设备
- **SDI帧：** 使用[zlobjects]表查找字节位置和位偏移
- **无岔区段：** SDCI和SDI帧都使用[zlobjects]表的bit_offset：
  - bit_offset = 0：使用低4位（位0-3）
  - bit_offset = 4：使用高4位（位4-7）
- 设备若位偏移不同（0或4）可共享同一字节
- 解析器自动处理大小端转换

### 重要：SDCI帧解析规则

**此规则对正确解码设备状态至关重要：**

#### 规则说明

```
SDCI帧解析步骤：
1. 使用[objects]表按对象索引查找设备
2. 获取设备状态值（1字节）
3. 使用[zlobjects]表的bit_offset提取正确的四位：
   - bit_offset = 0 → 提取低4位 (state_byte & 0x0F)
   - bit_offset = 4 → 提取高4位 ((state_byte & 0xF0) >> 4)
```

#### 重要性说明

许多区段共享SDI缓冲区中的同一字节：

- 字节331包含两个区段：
  - 位0-3（bit_offset=0）：54/56G（低四位）
  - 位4-7（bit_offset=4）：6/18G（高四位）

**使用错误四位将导致解码出错误的设备状态！**

#### 解析示例

**示例1：6/18G（使用高四位）**

lgxtq.zl条目：

```
[objects]
#,6/18G,204

[zlobjects]
#,6/18G,331,4
```

解析步骤：

1. 在SDCI帧中查找对象索引=204 → 6/18G
2. 获取state_byte = 0x30（示例值）
3. bit_offset = 4 → 提取高四位：
   - high_nibble = (0x30 & 0xF0) >> 4 = 0x3
4. 解码状态：high_nibble = 0x3表示"占用+锁闭"

**示例2：54/56G（使用低四位）**

lgxtq.zl条目：

```
[objects]
#,54/56G,203

[zlobjects]
#,54/56G,331,0
```

解析步骤：

1. 在SDCI帧中查找对象索引=203 → 54/56G
2. 获取state_byte = 0x03（示例值）
3. bit_offset = 0 → 提取低四位：
   - low_nibble = 0x03 & 0x0F = 0x3
4. 解码状态：low_nibble = 0x3表示"占用+锁闭"

**示例3：10号道岔区段（使用完整字节）**

lgxtq.zl条目：

```
[objects]
#,10,0

[zlobjects]
#,10,1,0
```

解析步骤：

1. 在SDCI帧中查找对象索引=0 → 10号道岔
2. 获取state_byte = 0x01（示例值）
3. bit_offset = 0 → 但道岔区段使用完整字节
4. 道岔区段通常使用低四位或解释完整字节

### 常见错误避免

| 错误做法                   | 正确做法                               |
| -------------------------- | -------------------------------------- |
| 将object_index用作字节位置 | object_index用于SDI；byte_index用于SDI |
| 始终使用低四位             | 检查zlobjects表的bit_offset            |
| 完全忽略bit_offset         | bit_offset决定使用哪个四位             |
| 混淆[objects]和[zlobjects] | [objects] → SDCI；[zlobjects] → SDI  |

### 四位状态编码参考

区段状态编码：

| 四位值 | 状态      | 说明           |
| ------ | --------- | -------------- |
| 0x0    | 空闲      | 无占用，无锁闭 |
| 0x1    | 锁闭      | 仅锁闭         |
| 0x2    | 占用      | 仅占用         |
| 0x3    | 占用+锁闭 | 既占用又锁闭   |

**注意：** 不同系统可能使用不同编码，请参考具体实现。

### 设备类型识别

| 名称模式 | 类型     | 示例              |
| -------- | -------- | ----------------- |
| 纯数字   | 道岔区段 | "10", "110"       |
| D+数字   | 信号机   | "D114", "D6"      |
| 其他     | 无岔区段 | "DK5", "102/104G" |

## 通信内容分类汇总

| 分类     | 方向      | 帧类型         | 目的         |
| -------- | --------- | -------------- | ------------ |
| 站场信息 | 联锁→CTC | SDI, SDCI, FIR | 状态数据     |
| 工作状态 | 双向      | RSR            | 系统状态     |
| 控制命令 | CTC→联锁 | BCC            | 设备控制     |
| 时钟同步 | 联锁↔CTC | TSQ, TSD       | 时间同步     |
| 心跳     | 双向      | ACK, DC2, DC3  | 连接健康     |
| 模式转换 | 联锁↔CTC | ACQ, ACA, RSR  | 控制模式变更 |
